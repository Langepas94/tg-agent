# Команды и память

## Команды пользователя

| Команда | Назначение |
| --- | --- |
| `/start <пароль>` | Получить доступ к боту |
| `/help` | Показать команды, доступные вашей роли |
| `/trip <запрос>` | Начать сценарий выбора и планирования отдыха |
| `/profile` | Показать сохраненный профиль |
| `/profile <ключ> <значение>` | Сохранить постоянное предпочтение |
| `/profile clear` | Очистить профиль |
| `/info <метка> <текст>` | Сохранить дополнительное правило, применяемое по ситуации |
| `/info clear` | Очистить дополнительную информацию |
| `/facts` | Показать факты, которые сохранил бот |
| `/compact` | Очистить короткий контекст и активный сценарий, сохранив профиль и долгосрочные факты |
| `/reset` | Сбросить память текущего диалога, сохранив долгосрочные факты |
| `/watches` | Показать доступные вам периодические наблюдения |
| `/unwatch <id>` или `/unwatch all` | Остановить свое наблюдение |

## Профиль и дополнительная информация

Для повторяющихся предпочтений используйте `/profile`. Поддерживаются ключи
`home_city`, `preferred_units`, `comfort_temp_min`, `comfort_temp_max`,
`dislikes_rain`, `interests` и `language`.

Примеры:

```text
/profile home_city Москва
/profile dislikes_rain true
/profile interests hiking, cycling
/info files Файлы нужны в формате .docx, имя должно содержать дату
```

Профиль учитывается постоянно. Запись `/info` используется только когда ее
метка относится к текущему запросу. Проверить сохраненные данные можно
командами `/profile`, `/info` и `/facts`.

## Команды владельца

Владелец дополнительно видит `/connect`, `/mcps`, `/tools`, `/call`, `/watch` и
`/disconnect`. Они управляют подключенными инструментами и не нужны обычному
пользователю. Сообщение о том, что команда доступна только владельцу, означает
ограничение роли, а не ошибку авторизации.
